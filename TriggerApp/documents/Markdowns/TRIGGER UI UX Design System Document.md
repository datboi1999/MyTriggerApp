### **TRIGGER UI/UX Design System Document**

This document establishes the visual language and user experience framework for **TRIGGER**, a high-performance "cyber-analytics" platform. The design system is optimized for dark environments, prioritizing high-contrast readability and real-time data density to support rapid campaign orchestration.

---

#### **1. Visual Vision & Design Language**
*   **Aesthetic Strategy:** A **"Dark Mode First"** cyber-analytics aesthetic.
*   **Core Motif:** The **"Trigger" motif**—a white lightning bolt within a blue square—symbolizing rapid, AI-powered growth.
*   **Thematic Elements:** Use of **neural network or mesh backgrounds** on splash screens to communicate the platform's AI-driven nature.
*   **UI Characteristics:** Implementation of **glassmorphism overlays**, neon highlights, and gradient-based engagement visualizations.

#### **2. Core Color Palette**
The palette is designed to minimize eye strain while making growth metrics "pop" against a deep background.

| Element | Color Name | Hex Code | Usage |
| :--- | :--- | :--- | :--- |
| **Primary Brand** | **Vibrant Blue** | `#0052FF` | Logo, primary actions (Login/Sign Up), "Growth Engine" buttons. |
| **Background (Core)** | **Deep Midnight Navy** | `#081120` | The primary canvas for all application screens. |
| **Secondary Surface** | **Navy Slate** | `#0F172A` | Secondary containers and background variations. |
| **Surface/Card** | **Dark Charcoal Grey** | — | Input fields, campaign cards, and analytics modules to create depth. |

| **Typography (Pri)** | **Pure White** | `#FFFFFF` | High-level headings (e.g., "TRIGGER", "Welcome Back"). |
| **Typography (Sec)** | **Light Slate/Grey** | — | Subtitles, placeholder text, and descriptive labels. |
| **Error State** | **Alert Red** | — | "Invalid URL" formats and critical error feedback. |

#### **3. Typography & Hierarchy**
*   **Primary Typeface:** **Inter** or **SF Pro Display** (Sans-Serif).
*   **Headings (H1/H2):** Bold and clean for screen titles and branding.
*   **Body Text:** Regular weight for high readability in data-dense areas like the Analytics Dashboard.
*   **Labels/Captions:** Uppercase or semi-bold small text for form labels (e.g., "EMAIL") and "AI Insight" tags.

#### **4. UI Components & Interaction**

##### **4.1 Buttons & Controls**
*   **Primary Buttons:** High-contrast blue with rounded corners and white text. These feature a **subtle blue outer glow** to give a "powered-on" feel.
*   **Secondary/Outline:** Transparent with thin borders for secondary actions like "Edit Setup".
*   **Start Trigger Action:** Enhanced with neon glows to emphasize the core utility of the web app.

##### **4.2 Input & Feedback**
*   **Form Fields:** Darkened containers with subtle grey borders and **left-aligned icons** (e.g., mail icon for email). Includes secure toggles for password visibility.
*   **Progress Indicators:** **Linear blue progress bars** for multi-step onboarding and platform contribution charts.
*   **Verification:** Blue/white checkmarks indicate verified account connections.

##### **4.3 Elevation & Depth**
*   **Layering:** Modules like "Secure Payment" use a slightly lighter grey surface to sit above the primary background.
*   **Glow Effects:** Used sparingly on primary metrics and "active" triggers to signify live system activity.

#### **5. Analytics & Data Visualization**
The UX is optimized for a **compact, data-dense layout** that allows users to monitor live engagement activity.
*   **AI Activity Feed:** A live, high-frequency event stream showing trigger executions and AI optimization decisions.
*   **Growth Trends:** Interactive charts (using **Recharts**) for follower and stream growth velocity.
*   **Real-Time Refresh:** Metrics must refresh every **≤3 seconds** for active campaigns to maintain the "live engine" feel.

#### **6. UX Principles for Growth**
*   **Speed to Launch:** The UI must be streamlined to allow users to launch growth campaigns in **under 3 minutes**.
*   **Clarity of Attribution:** UX should clearly track growth attribution by platform (TikTok, Spotify, etc.).
*   **Retention Visibility:** Dedicated UI indicators for the **Anti-Fluctuation Logic** to show users when the system is actively stabilizing their growth.
*   **Mobile-Friendly Targets:** While browser-based, elements should be sized for high-performance interaction.
